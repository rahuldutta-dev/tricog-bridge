Step 3 — Device Identity (Location Slug)

Assign a unique URL slug for this device. The source device or communicator pushes raw ECGs to its own per-device URL. No identity field is read from the payload.

URL-safe identifier · [A-Za-z0-9_-]{2,32} · must be unique within this Bridge install

Per-device Ingest URL

https://bridge.local:8444/push/RECEPTION_A

Configure your ECG machine or communicator to POST raw ECGs to this exact URL. Backend route is a single /push/:location handler — the path parameter is the device identity.

Payloads to an unknown slug return HTTP 404, are written to the audit log with the unknown slug and source IP, and surface as an alert in the Devices list.
